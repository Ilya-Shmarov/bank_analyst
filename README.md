# competitor-scanner — конкурентный анализ премиум-банкинга

Сервис сканирует открытые источники по премиальным пакетам банков РФ и
экосистемным подпискам, собирает данные в единый Excel-отчёт и ведёт
changelog изменений между сканами. Запуск ручной, из терминала.

## Что сканируется

**Наша линейка (база сравнения):** СберПремьер (уровни 1–3), СберПервый
(уровни 4–5), Sber Private Banking (уровень 6).

**Прямые конкуренты:** Т-Банк (Premium Bronze/Silver/Gold, Private Diamond),
Альфа-Банк (Alfa Only 4 уровня, A-Club), ВТБ (Привилегия 4 уровня, Прайм+
4 уровня), Газпромбанк (Премиум 3 уровня, Private), Озон Банк (Ultra
Bronze/Silver/Gold/Platinum), Райффайзен (Premium, 3 варианта входа).

**Lifestyle-конкуренты:** Яндекс Плюс, Ozon Premium, Wildberries Клуб.

**Источники (в порядке приоритета):** premiumbanking.info (агрегатор,
страницы по каждому уровню каждого банка) → официальные сайты банков →
страницы подписок. Frank RG и корень premiumbanking.info сохраняются
raw-снимками для аудита.

## Установка

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# Опционально — для сайтов с JS-рендерингом (Альфа-Банк, Ozon и т.п.):
.venv/bin/pip install playwright
.venv/bin/playwright install chromium
```

Без Playwright сервис работает, но JS-сайты будут помечены как
`js_required` и их поля останутся «не найдено».

## Запуск

```bash
.venv/bin/python main.py --scan-all               # полный скан
.venv/bin/python main.py --scan-bank tbank        # один банк (id — см. --list-sources)
.venv/bin/python main.py --scan-lifestyle         # только экосистемные подписки
.venv/bin/python main.py --list-sources           # список источников и id
```

Поведение при скане:

- прогресс по каждому источнику логируется в консоль (успех/ошибка/JS);
- уважается `robots.txt`, между запросами пауза 2.5 с (см. `REQUEST_PAUSE`
  в `scanner/sources.py`);
- недоступный источник не роняет скан — он помечается в метаданных, скан
  продолжается;
- в конце — сводка: сколько полей заполнено, сколько изменений найдено.

## Результаты

| Артефакт | Что это |
|---|---|
| `output/competitor_analysis.xlsx` | Отчёт: «Сводная» (тиры, сгруппированные по сегментам капитала 0–3 / 3–10 / 10–25 / 25–100 млн ₽), лист детализации на каждый банк, «Lifestyle-конкуренты», «Изменения» (changelog было/стало с датами), «Метаданные» |
| `data/history.json` | Полная история сканов (последние 20) + накопленный changelog |
| `data/raw/<дата>/<источник>.html` | Сырые снимки страниц для аудита (в git не коммитятся) |

Excel-файл регенерируется при каждом запуске из `data/history.json` —
история изменений и данные прошлых сканов при этом сохраняются, а
форматирование остаётся консистентным.

При точечном скане (`--scan-bank`, `--scan-lifestyle`) данные по остальным
банкам подтягиваются из последнего скана, чтобы сводная таблица оставалась
полной и diff не показывал ложные «тир пропал».

## Принципы данных

- **Ничего не выдумываем.** Поле заполняется только цитатой из источника;
  если данных нет — в ячейке «не найдено».
- **Дата скана фиксируется** рядом с данными каждого тира — премиальные
  тарифы (особенно ставки) меняются часто.
- **Антибот не обходим.** Если сайт блокирует запросы (403/капча), источник
  помечается недоступным. Playwright используется только как рендер
  обычных страниц, не для обхода защит.

## Как добавить банк или подписку

1. В `scanner/sources.py` добавьте запись в `BANKS`: id, name, type
   (`bank` / `lifestyle`), список тиров с candidate-URL (первый успешный
   выигрывает) и сегментом капитала.
2. При необходимости зарегистрируйте специализированный парсер в
   `scanner/parse.py` → `PARSER_REGISTRY["<bank_id>"] = MyParser()`.
   По умолчанию работает `GenericParser` (извлечение цитат по ключевым
   словам полей).

Ядро (`fetch` / `diff` / `excel_writer`) при этом не меняется.

## Структура

```
├── main.py                  # CLI: --scan-all / --scan-bank / --scan-lifestyle
├── scanner/
│   ├── sources.py           # реестр банков, тиров, URL, полей и сегментов
│   ├── fetch.py             # requests + robots.txt + rate limit + Playwright-fallback
│   ├── parse.py             # GenericParser (цитаты по ключевым словам) + реестр парсеров
│   └── diff.py              # история сканов, diff, merge точечных сканов
├── report/
│   └── excel_writer.py      # генерация xlsx (5 типов листов)
├── data/                    # history.json + raw-снимки
└── output/                  # competitor_analysis.xlsx
```

## Известные ограничения

- `GenericParser` извлекает **цитаты**, а не нормализованные числа —
  для точного сравнения ставок нужен специализированный парсер на банк
  (архитектурно предусмотрено через `PARSER_REGISTRY`).
- Diff чувствителен к любому изменению текста страницы — возможны
  «шумные» изменения при редизайне источника.
- Сегмент капитала тира — конфигурация раскладки в `sources.py`
  (сверена с порогами входа на июль 2026), а не данные из источника.
