# competitor-scanner — конкурентный анализ премиум-банкинга

Сервис сканирует открытые источники по премиальным пакетам банков РФ и
экосистемным подпискам, сливает данные из нескольких источников с фиксацией
провенанса каждой цифры, считает собственную сравнительную оценку пакетов
и ведёт changelog изменений. Запуск ручной, из терминала.

## Что сканируется

**Наша линейка (база сравнения):** СберПремьер (уровни 1–3), СберПервый
(уровни 4–5), Sber Private Banking (уровень 6).

**Прямые конкуренты:** Т-Банк (Premium Bronze/Silver/Gold, Private Diamond),
Альфа-Банк (Alfa Only 4 уровня, A-Club), ВТБ (Привилегия 4 уровня, Прайм+
4 уровня), Газпромбанк (Премиум 3 уровня, Private), Озон Банк (Ultra
Bronze/Silver/Gold/Platinum), Райффайзен (Premium, 3 варианта входа).

**Lifestyle-конкуренты:** Яндекс Плюс, Ozon Premium, Wildberries Клуб.

**Международные digital-first банки** (отдельный блок в Сводной, та же
схема атрибутов): Revolut (Premium / Metal / Ultra), N26 (Go экс-You /
Metal), Wise (без тиров — multi-currency сценарий), Monzo (Extra / Perks /
Max — актуальная линейка, заменившая Plus/Premium в 2024). Все поля
заполнены верифицированными фактами с официальных страниц тарифов и Help
Centre (source_url + date_checked у каждого значения); неприменимые
категории — прочерк «— (не предусмотрено моделью продукта)», статуса
«не найдено» в этом блоке нет. Цены в валюте страны (курсы ЦБ на дату
скана — на листе «Методика»); в балльную оценку блок не входит.

## Источники и их слияние

У каждого тира несколько независимых источников (см. `scanner/sources.py`):

| Источник | Роль | Приоритет |
|---|---|---|
| Ручная проверка (`scanner/curated.py`) | верифицированные факты со ссылкой на первоисточник и датой проверки | 0 (высший) |
| Официальный сайт банка | первоисточник условий, тарифов, ставок | 1 |
| premiumbanking.info (ПБИ) | структурированные страницы каждого уровня | 2 |
| Banki.ru / Sravni.ru / Bankiros | кросс-проверка | 3–5 |
| Frank RG | рыночный контекст (raw-снимок) | 6 |

Правила слияния (`scanner/merge.py`):

1. Верифицированный вручную факт всегда приоритетнее автопарсинга.
2. Среди автоматических источников сначала сравнивается **качество**
   (структурированные данные > цитаты-сниппеты), внутри одного качества —
   приоритет источника (официальный сайт первый).
3. Значения остальных источников не выбрасываются — хранятся в
   `alternatives`. Если числа расходятся с основным значением, поле
   помечается `divergent` и попадает в колонку «Расхождение источников».

Каждое значение в отчёте несёт `[источник: …, проверено: дата]`.

**Замечание по кросс-источникам:** robots.txt Banki.ru запрещает обкачку
их страниц — сервис уважает запрет и помечает источник как недоступный.
Sravni.ru и Bankiros дают страницы уровня банка (не конкретного пакета),
поэтому работают только как слабая кросс-проверка.

## Собственная оценка пакетов (не рейтинг ПБИ)

`scanner/scoring.py`: по каждой категории привилегий из текста данных
извлекается метрика (визиты в БЗ, % кэшбэка, ставка вклада, покрытие ВЗР…),
метрика переводится в балл 0–5 по пороговым таблицам, итог — взвешенная
сумма (веса в `WEIGHTS`, сумма = 1.0, правится без изменения кода расчёта).

Формула, веса, пороги и разбивка балла каждого тира по категориям — на
листе **«Методика оценки»** отчёта: балл воспроизводим вручную.
Оценка ПБИ «ценность пакета в год» хранится отдельным справочным полем
и в расчёте не участвует.

## Верифицированные факты (curated)

`scanner/curated.py` — слой фактов, проверенных вручную по первоисточникам
(сейчас — блок Сбера: консьерж по уровням, кэшбэк и курсы обмена бонусов,
вклады, автоуслуги). Это не разовый хардкод: записи применяются при каждом
скане, несут `source_url` и `date_checked`; записи старше 90 дней
помечаются «проверить актуальность». Устарело — обновите значение и дату
или удалите запись, поле вернётся к автопарсингу.

## Установка

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# Опционально — для сайтов с JS-рендерингом (Альфа-Банк, sberpb.ru и т.п.):
.venv/bin/pip install playwright
.venv/bin/playwright install chromium
```

## Запуск

```bash
.venv/bin/python main.py --scan-all               # полный скан
.venv/bin/python main.py --scan-bank tbank        # один банк (id — см. --list-sources)
.venv/bin/python main.py --scan-lifestyle         # только экосистемные подписки
.venv/bin/python main.py --build-digest           # HTML-дайджест изменений за 60 дней
.venv/bin/python main.py --list-sources           # список источников и id
```

## Дайджест изменений (output/digest.html)

Дайджест **пересобирается автоматически в конце каждого скана** (`--scan-all`,
`--scan-bank`, `--scan-lifestyle`) — отдельная команда не нужна.
`--build-digest` оставлена для ручной пересборки без скана (например, после
правки `digest/rules.json`).

**В дайджест попадают только рыночные изменения.** Разделение — на уровне
данных (scanner/diff.py), а не фильтром постфактум:

| Лог | Что содержит | Куда идёт |
|---|---|---|
| `data/market_changes.json` | Изменения условий продукта: оба значения были фактически найдены и содержательно отличаются (цена, ставка, лимит, состав привилегий) | Дайджест |
| `data/service_log.json` | Техника сервиса: дозаполнение пустых полей («не найдено» → значение), пропажа данных из источника, статусы источников, правки схемы/реестра, переформулировки без изменения цифр | Только отладка |

Дозаполнение ранее пустого поля НЕ создаёт записи в дайджесте — это
исправление нашей базы, а не новость рынка. Полный техлог изменений
остаётся на листе «Изменения» Excel-отчёта.

Карточки: банк → тир → что изменилось → было/стало → дата → ссылка на
источник; категории — запуск / изменение условий / маркетинг / сокращение
привилегии; свежие сверху; правила классификации — `digest/rules.json`.
Опционально `--use-ai-summary` — формулировки через Anthropic API
(claude-opus-4-8, нужен `ANTHROPIC_API_KEY`); без флага — шаблонные.

Поведение: прогресс по каждому источнику в консоль; robots.txt уважается;
пауза 2.5 с между запросами; общие URL кэшируются на время скана;
недоступный источник не роняет скан; антибот/капча не обходятся
(Playwright используется только как рендер обычных страниц).

## Результаты

| Артефакт | Что это |
|---|---|
| `output/competitor_analysis.xlsx` | Сводная (сегменты капитала + итоговый балл + расхождения), детализация по банкам, Lifestyle, Изменения (с источником нового значения; ручные уточнения помечены), Методика оценки, Метаданные |
| `data/history.json` | История сканов (последние 20) + накопленный changelog |
| `data/raw/<дата>/<тир>__<источник>.html` | Сырые снимки страниц для аудита (в git не коммитятся) |

Excel регенерируется при каждом запуске из `data/history.json` — история
не теряется. При точечном скане данные остальных банков подтягиваются из
последнего скана.

## Как добавить банк, подписку или источник

1. Банк/подписка: запись в `BANKS` (`scanner/sources.py`) — тиры, сегмент
   капитала, список источников с candidate-URL.
2. Источник: добавьте `_src("<source_id>", url)` в `sources` тира или
   `extra_sources` банка; при новом типе источника — запись в `SOURCE_META`.
3. Специализированный парсер: `PARSER_REGISTRY["<bank_id>"] = MyParser()`
   в `scanner/parse.py` (по умолчанию GenericParser — цитаты по ключевым
   словам; для ПБИ работает структурированный dt/dd-парсер).
4. Верифицированный факт: запись в `CURATED_FACTS` (`scanner/curated.py`)
   с `source_url` и `date_checked`.

Ядро (fetch / merge / diff / scoring / excel) при этом не меняется.

## Принципы данных

- Ничего не выдумываем: поле = цитата/факт из источника, иначе «не найдено».
- Дата проверки фиксируется рядом с каждым значением: премиальные тарифы
  (особенно ставки вкладов) меняются вслед за ключевой ставкой ЦБ.
- Антибот не обходим: 403/капча → источник помечен недоступным.
- Расхождения источников не замалчиваются — выносятся в отдельную колонку.

## Структура

```
├── main.py                  # CLI: --scan-all / --scan-bank / --scan-lifestyle
├── scanner/
│   ├── sources.py           # реестр банков, тиров, источников, полей, сегментов
│   ├── curated.py           # верифицированные вручную факты (source_url + date_checked)
│   ├── fetch.py             # requests + robots.txt + rate limit + кэш + Playwright-fallback
│   ├── parse.py             # PBI-парсер (dt/dd) + GenericParser + реестр парсеров
│   ├── merge.py             # слияние источников, провенанс, детект расхождений
│   ├── scoring.py           # собственная оценка: веса, пороги, разбивка
│   └── diff.py              # история сканов, diff, changelog
├── report/
│   └── excel_writer.py      # генерация xlsx (6 типов листов)
├── data/                    # history.json + raw-снимки
└── output/                  # competitor_analysis.xlsx
```

## Известные ограничения

- GenericParser извлекает цитаты, а не нормализованные числа — скоринг
  работает по извлечённым метрикам и показывает их явно в разбивке.
- Diff чувствителен к изменению текста страницы — при редизайне источника
  возможны «шумные» изменения.
- Сегмент капитала тира — конфигурация раскладки (сверена с порогами входа
  на июль 2026), а не данные из источника.
- ozon.ru и wildberries.ru закрыты антиботом; frankrg.com рвёт соединение —
  фиксируются как недоступные, без попыток обхода.
